Build **Horizon** — a sky gradient generator with atmospheric controls, glass UI, and PNG/SVG export. This is a standalone React + TypeScript application. Vite for build tooling.

**GRADIENT ENGINE**

The gradient is rendered as a full-viewport `<canvas>` element behind all UI. It must be redrawn on every parameter change using `requestAnimationFrame` — no CSS gradients, canvas only. This gives pixel-level control and enables export at arbitrary resolution.

Implement `buildSkyGradient(params: SkyParams): ImageData` as the core rendering function. It constructs a multi-stop vertical linear gradient by sampling a continuous color function at `n` points across the canvas height. The color function takes a normalized Y position `(0 = top, 1 = bottom)` and the full `SkyParams` object and returns an `[r, g, b]` tuple.

The color function must be atmospheric — not a simple lerp between two colors but a physically-inspired model:

- **Zenith zone** (Y: 0.0 → 0.35): Deep sky color, slightly desaturated and shifted toward violet at high altitude. Controlled by `zenithHue`, `zenithSaturation`, `zenithLightness`.
- **Mid sky zone** (Y: 0.35 → 0.62): Transition zone. Color blends using a smooth cubic easing function — `t³(6t² − 15t + 10)` — not linear interpolation. This is where the most color complexity lives.
- **Horizon zone** (Y: 0.62 → 0.82): The critical band. Warm colors, glow, atmospheric haze. Controlled by `horizonHue`, `horizonIntensity`, `hazeWidth`.
- **Ground zone** (Y: 0.82 → 1.0): Silhouette / ground color, dark, slightly warm.

All zone boundaries are soft — use the smoothstep function `f(t) = t²(3 − 2t)` for all transitions between zones. No hard edges anywhere.

**Atmospheric presets** — implement these as named `SkyParams` configurations selectable from the control panel:

- `GOLDEN_HOUR`: Amber horizon (#E8A020), violet zenith (#1A0A3C), high haze intensity
- `BLUE_HOUR`: Steel blue zenith (#0D1B3E), pale cyan horizon (#A8C8E8), low saturation
- `OVERCAST`: Near-white zenith (#D8DDE8), grey-blue horizon (#8090A8), zero haze
- `STORM`: Dark grey-purple zenith (#1A1520), olive-grey horizon (#4A4830), heavy atmosphere
- `DUSK`: Deep indigo zenith (#0A0818), coral-pink horizon (#E85030), dramatic transition
- `CLEAR_NOON`: Saturated blue zenith (#1060C8), pale blue horizon (#A8C8F0), minimal haze

**EFFECTS LAYER**

Render effects on a second `<canvas>` element layered above the gradient canvas, below the UI. Effects are composited with `globalCompositeOperation: 'screen'` at low opacity so they illuminate rather than obscure.

**Noise effect**: Implement smooth Perlin-like noise using a summed sinusoid approximation — no external libraries. The noise canvas renders a full-viewport texture of soft luminance variation at `opacity: noiseOpacity` (range 0–0.15). Grain size controlled by `noiseScale` — small values produce fine film grain, large values produce soft cloud-like variation. Noise is static (computed once) unless `noiseAnimated` is true, in which case it drifts slowly using a time offset in the `requestAnimationFrame` loop.

**Glass bars effect** (as shown in the reference): Vertical bars of slightly lighter luminance, soft-edged, semi-transparent. Rendered as a series of vertical `linearGradient` fills on the effects canvas. Each bar: width `barWidth`px, horizontal spacing `barSpacing`px, opacity `barOpacity` (range 0–0.12). Bar edges use a Gaussian falloff — brightest at center, fading to zero at edges. Bar positions are deterministic from a seed value so they don't shift on re-render. Number of bars fills the viewport width based on spacing. This should reproduce the effect in the reference image — subtle vertical light columns suggesting glass or atmospheric refraction.

Effects parameters:
```typescript
interface EffectsParams {
  noiseEnabled: boolean
  noiseOpacity: number      // 0–0.15
  noiseScale: number        // 10–200
  noiseAnimated: boolean
  barsEnabled: boolean
  barOpacity: number        // 0–0.12
  barWidth: number          // 20–120px
  barSpacing: number        // 40–200px
  barSeed: number           // deterministic positioning
}
```
